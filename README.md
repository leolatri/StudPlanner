# СтудПланнер

**СтудПланнер** — мобильное приложение, цель которого облегчить студенческую жизнь, объединив все электронные ресурсы в одном месте: расписание, контакты, учебные материалы и профиль пользователя.

Приложение создаётся на основе заранее подготовленного дизайна в Figma; возможные совпадения с другими продуктами являются случайными.

## Идея и польза

У студентов обычно много разрозненных источников: сайт/портал университета, таблицы с расписанием, чаты, почта, файлообменники и т.д. СтудПланнер собирает ключевые вещи в одном приложении, чтобы:

- Быстро находить расписание по своей группе.
- Иметь под рукой контакты преподавателей и одногруппников.
- Хранить и открывать учебные материалы в электронном виде.
- Управлять базовыми данными профиля без лишних сервисов.

## Разделы приложения

### Профиль

Раздел для управления учётной записью и личными данными.

Функции:
- Просмотр данных профиля.
- Редактирование существующих данных.
- Изменение логина и пароля.
- Управление почтой (просмотр/редактирование привязанного email).

### Расписание

Раздел, где пользователь видит расписание занятий, привязанное к выбранной группе.

Сценарий работы:
1. Пользователь открывает экран «Расписание».
2. В фильтре выбирает нужную группу.
3. Получает расписание по дням (например, «Пн–Вс» или «Неделя 1/Неделя 2» — как заложено дизайном).

Функции:
- Фильтр/выбор группы.
- Просмотр расписания по дням.
- Быстрый переход между днями.

### Контакты

Справочник контактов по выбранной группе: преподаватели и студенты.

Данные в карточке контакта:
- ФИО.
- Почта.
- Номер телефона (если он указан/доступен).

Функции:
- Список преподавателей группы.
- Список студентов группы.
- Быстрый поиск/фильтрация (опционально, но очень удобно).
- Быстрые действия: написать на почту (mailto), позвонить (tel) — если есть телефон.

### Библиотека

Раздел с учебными материалами университета в электронном виде.

Функции:
- Просмотр материалов.
- Загрузка (добавление) нового материала: материал будет доступен **локально** на устройстве (виден только пользователю).
- Скачивание материала на устройство (в файл/память устройства), чтобы открыть оффлайн.

Примечания по логике:
- Если материал добавлен пользователем, он хранится локально и не обязательно появляется у других.
- Для файлов полезно предусмотреть: тип (pdf/docx/ссылка), размер, дата добавления, предмет/категория.

## Запуск приложения

Ниже — базовые команды запуска.

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

```bash
npm run start
```

После запуска откроется dev-сервер (Expo), а приложение можно запустить на устройстве или браузере.

### Сборка APK (Android)

Команда сборки `build:apk`:

```bash
npm run build:apk
```
