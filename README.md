# СтудПланнер (StudPlanner)

Мобильное приложение для студентов, объединяющее расписание, контакты,
учебные материалы и профиль в одном месте.

## О проекте

Студенты ежедневно пользуются множеством разрозненных ресурсов: сайт вуза,
таблицы с расписанием, чаты, почта, файловые хранилища. **СтудПланнер**
решает эту проблему, собирая ключевые сервисы в едином интерфейсе:

- Быстрый просмотр расписания по своей группе
- Все контакты преподавателей под рукой
- Хранение и просмотр учебных материалов на устройстве
- Управление личным профилем без лишних сервисов

## Технологический стек

### Backend (серверная часть)
- **Язык:** Python 3
- **Фреймворк:** FastAPI
- **База данных:** SQLite + SQLAlchemy
- **Аутентификация:** JWT (JSON Web Tokens)
- **Документация API:** Swagger UI (автоматически доступен после запуска)

### Frontend (мобильное приложение)
- **Платформа:** Expo / React Native
- **Язык:** TypeScript
- **Навигация:** React Navigation
- **Управление состоянием:** MobX-State-Tree
- **Стилизация:** централизованные стили

## Модели данных (DTO)

В приложении используются следующие типизированные объекты передачи данных (Data Transfer Object). Они описывают структуру информации, которой обмениваются клиент и сервер.

### Диаграмма классов

Ниже представлена UML-диаграмма, показывающая все DTO и связи между ними.

<img width="1323" height="855" alt="image" src="https://github.com/user-attachments/assets/9bba04f0-eb84-42b0-b5e6-3a6538bc2829" />


### Описание сущностей

| DTO | Назначение | Основные поля |
|-----|------------|----------------|
| **BookDTO** | Учебный материал (книга, статья, ссылка) | `id`, `name`, `autors`, `url`, `owner_id` (кто добавил) |
| **FeedbackDTO** | Отзыв о преподавателе | `id`, `text`, `grade` (оценка), `autor`, `isPersonal` (виден ли только владельцу) |
| **ContactDTO** | Контакт (преподаватель или студент) | `id`, `fio`, `email`, `uniSubjects` (дисциплины), `feedbacks` (список отзывов) |
| **SubjectDTO** | Занятие в расписании | `id`, `type` (лекция/практика), `name`, `room`, `duration`, `timeAndDate`, `groups` (ID групп) |
| **GroupDTO** | Учебная группа | `id`, `name`, `isActive` (активна ли группа), `isSelected` (выбрана ли пользователем) |
| **UserDTO** | Пользователь системы | `id`, `email`, `first_name`, `second_name`, `telegram`, `phone_number`, `is_admin` |
| **AdditionDataDTO** | Контейнер для загрузки всех данных приложения | `library`, `groupList`, `contacts`, `subjects` – массивы соответствующих DTO |

### Примечания по связям

- **ContactDTO владеет** массивом `FeedbackDTO` (композиция): при удалении контакта удаляются и его отзывы.
- **BookDTO.owner_id** – опциональная ссылка на `UserDTO.id` (книга может быть общей).
- **SubjectDTO.groups** – массив строковых идентификаторов, указывающих на `GroupDTO.id` (занятие может принадлежать нескольким группам).
- **AdditionDataDTO** используется для первичной загрузки данных при старте приложения и синхронизации.

> Все массивы (`[]`) могут быть пустыми, если данные отсутствуют. Поля, помеченные как `| null`, допускают значение `null` на уровне БД.


## Установка и запуск

### Предварительные требования

- Node.js (рекомендуется LTS-версия)
- npm или Yarn
- Python 3.8+ и пакетный менеджер pip
- Установленный Expo CLI (опционально, для разработки на устройстве)

### 1. Клонирование репозитория

```bash
git clone https://github.com/leolatri/StudPlanner.git
cd StudPlanner
```

### 2. Запуск Backend

```bash
cd backend
pip install -r requirements.txt   # установите зависимости (если есть requirements.txt)
# или установите вручную: pip install fastapi uvicorn sqlalchemy pyjwt ...
python init_db.py                # инициализация таблиц БД
python seed.py                   # (опционально) наполнение тестовыми данными
uvicorn main:app --reload        # запуск сервера с автообновлением
```

После запуска сервер будет доступен по адресу `http://localhost:8000`.
Интерактивная документация API — `http://localhost:8000/docs`.

### 3. Запуск Frontend

```bash
cd frontend
npm install
npm run start
```

Expo запустит dev-сервер. Для просмотра приложения:

- Отсканируйте QR-код через приложение Expo Go на своём устройстве
- Или запустите эмулятор Android/iOS
- Или откройте в браузере (Web-режим)

## Сборка APK (Android)

Из директории `frontend` выполните:

```bash
npm run build:apk
```

Для сборки через EAS Build (рекомендуется) убедитесь, что файл `eas.json`
настроен правильно, и выполните:

```bash
eas build --platform android
```

## API (основные endpoint'ы)

| Метод | Путь                        | Описание                         |
|-------|-----------------------------|----------------------------------|
| POST  | `/auth/login`               | Вход по логину/паролю            |
| GET   | `/users/me`                 | Данные текущего пользователя     |
| GET   | `/groups`                   | Список групп                     |
| GET   | `/schedule?group_id={id}`   | Расписание группы                |
| GET   | `/contacts?group_id={id}`   | Контакты (студенты/преподаватели)|
| GET   | `/books`                    | Список учебных материалов        |
| POST  | `/books`                    | Добавить материал                |

Полный список эндпоинтов доступен после запуска сервера по адресу
`http://localhost:8000/docs`.
